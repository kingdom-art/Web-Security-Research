# Web-Security-Research
Notes, practice exercises, and workflows related to web application security testing, reconnaissance techniques, and penetration testing.

Web Security Practice – Recon Workflow

Today I practiced basic web application reconnaissance techniques on an in-scope asset.

Steps performed
	•	Checked robots.txt for disallowed paths
	•	Reviewed sitemap.xml to identify accessible pages
	•	Tested common directories and endpoints
	•	Performed basic parameter testing
	•	Inspected network requests using browser DevTools

Outcome

No vulnerabilities were discovered during this exercise. However, the session helped reinforce the importance of structured recon and evaluating the available attack surface before deeper testing.

Key takeaway

Effective bug hunting relies heavily on methodical reconnaissance and persistence.

Real bug hunting isn’t “find a bug every night.” It’s more like:
	•	80% recon & testing
	•	15% dead ends
	•	5% actual vulnerabilities
